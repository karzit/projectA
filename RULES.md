# 게임 규칙

## 기본 구조

- **덱**: 카드 15장. 게임 시작 시 **전부 패**에 들고 시작한다. 필드는 비어 있다.
- **비용 없음**: 카드를 내는 데 별도의 자원(마나 등)이 필요하지 않다.
- **패배 조건**: **턴 종료 시 필드가 비어 있으면** 진다.

---

## 진행 순서

### 오프닝 페이즈

- 양측이 번갈아 가며 **최대 3장**을 필드에 낼 수 있다. 각 카드는 배치할 **셀 번호**를 지정한다.
- 유닛은 즉시 필드에 배치되어 이후 카드의 **배경 조건**을 충족시킬 수 있다.
- 오프닝 중 발동되는 **전개·효과는 보류**된다 — 양측이 모두 오프닝을 마칠 때 A→B 순서로 일괄 처리된다.
- 3장을 모두 내거나 `finishOpening`을 선언하면 오프닝 종료.
- 양측 오프닝이 끝나면 보류 효과 처리 → 강제 능력 정산 → **A 선턴 메인 페이즈** 시작.

### 메인 페이즈

- 한 턴에 **원하는 만큼** 카드를 낼 수 있다 (제한 없음).
- 카드 효과(`onPlay`)는 **턴 종료(pass) 시** 낸 순서대로 일괄 처리된다.
  - **`개입` 키워드** 카드는 예외 — 낸 즉시 효과가 처리된다.
  - 유닛 소환(필드 배치)은 항상 즉시.
- **각 유닛은 공격 또는 이동 중 하나만** 가능하다 (`actedThisTurn`).
- **패스**는 "이번 턴 끝"을 선언하는 행동이다. 카드를 내지 않고 바로 패스해도 된다.

---

## 전장 그리드

플레이어당 **9칸**: **전열**(0–4, 5슬롯) + **후열**(5–8, 4슬롯). 셀당 유닛 1개 최대.

```
전열:  [0] [1] [2] [3] [4]
후열:  [5] [6] [7] [8]
```

**인접 관계** (이동·협공 판정):

```
0↔1,5 / 1↔0,2,5,6 / 2↔1,3,6,7 / 3↔2,4,7,8 / 4↔3,8
5↔0,1,6 / 6↔1,2,5,7 / 7↔2,3,6,8 / 8↔3,4,7
```

**공격 범위** (기본, 상대 전열 대상):

| 공격 셀 | 대상 셀 |
|---------|---------|
| 전열 0 | 0, 1 |
| 전열 1 | 0, 1, 2 |
| 전열 2 | 1, 2, 3 |
| 전열 3 | 2, 3, 4 |
| 전열 4 | 3, 4 |
| 후열 5 | 0, 1 |
| 후열 6 | 1, 2 |
| 후열 7 | 2, 3 |
| 후열 8 | 3, 4 |

**이동**: 행동권(`actedThisTurn`)이 남은 유닛이 **인접 빈 셀**로 이동. 이동도 행동권 소모.

## 카드의 종류

| 종류 | 설명 |
|------|------|
| **유닛** | 필드에 배치되어 전투에 참가한다. 힘·지혜·지략을 가진다. |
| **주문** | 낸 즉시 효과가 발동되고 소멸한다. 필드에 남지 않는다. (`개입` 키워드) |

---

## 능력치 (스탯)

| 스탯 | 이름 | 설명 |
|------|------|------|
| **힘** | 전투력 | 전투 시 높은 쪽이 낮은 쪽을 파괴. 동점이면 양쪽 모두 파괴. |
| **지혜** | 임계치 조건 | 소모되지 않음. 내 진영의 지혜 합계가 조건을 충족해야 특정 카드를 낼 수 있음. |
| **지략** | 방어 임계치 | 상대의 지혜 조건 카드에 자동 봉쇄(반응) 가능. |

스탯은 효과에 의해 **변동 가능**하다 (강화·교환·일시 버프 등).

---

## 배경 (배경 조건)

카드를 낼 때 **필요한 조건**. 내는 순간에만 확인하며, 이후에는 무시된다.

| 조건 종류 | 예시 |
|-----------|------|
| 특정 유닛 존재 | 필드에 '돌원숭이'가 있어야 한다 |
| 환경 항목 존재 | 지형이 '산'이어야 한다 |
| 지혜 임계치 | 내 진영의 지혜 합계 ≥ 15 |
| 강한 유닛 없음 | 내 진영에 힘 7 이상 유닛이 없어야 한다 |
| 키워드 보유 유닛 | 특정 키워드를 가진 유닛이 존재해야 한다 |
| 묘지 조건 | 사망한 유닛 중 특정 키워드를 가진 카드가 있어야 한다 |

---

## 환경 (환경 전개)

- 카드가 **전개** 효과로 환경을 바꿀 수 있다 (예: 지형:산, 지역:왕성 등).
- **같은 타입의 환경은 중복되지 않는다** — 새 값이 덮어쓴다.
- 서로 다른 타입은 동시에 공존한다.
- 환경은 **전역(글로벌)**으로 공유된다.

## 전투

### 1:1 전투 (기본)

1. 공격 유닛과 수비 유닛의 **힘을 비교**한다.
2. **힘이 낮은 유닛이 파괴**된다.
3. **힘이 같으면 양쪽 모두 파괴**된다.

### 협공 (다수 수비)

수비 측은 공격 선언에 대해 **여러 유닛으로 함께 방어**할 수 있다.

1. 공격자가 **공격 대상 유닛**을 지정한다.
2. 수비 측은 **주 수비 유닛의 인접 셀** 유닛을 추가 협공자로 지정할 수 있다.
3. 협공 참여 유닛 전체의 **힘을 합산**해 공격자의 힘과 비교한다.
4. **수비 합산 ≥ 공격자** → 공격자·수비 유닛 **전원 생존** (공격 막힘)
5. **수비 합산 < 공격자** → 수비 유닛 **전원 파괴**, 공격자 생존

> 협공에 참여한 유닛은 **그 턴 동안 다른 공격에 협공 참여 불가** (`blockedThisTurn`).
> 단, 공격 대상이 되는 것은 가능하다.

> `이중방어` 키워드를 가진 유닛은 한 턴에 협공을 **2회** 할 수 있다.

> `공격 불가` 키워드를 가진 유닛은 공격에 나설 수 없다.

> `협력불가` 키워드를 가진 유닛은 협공 블로커가 될 수도, 협공 방어를 받을 수도 없다.

> `전투면역` 키워드를 가진 유닛은 전투로 파괴되지 않는다 (효과 파괴는 적용됨).

### 타겟팅 리액션

공격·주문이 대상을 결정할 때 다음 파이프라인을 거친다:

- **호위** 키워드: 수비 측에 `호위` 유닛이 있으면 다른 무작위 아군이 대신 대상이 된다.
- **지략 opt-in**: 지혜 조건 카드가 플레이될 때 봉쇄 가능한 `지략` 유닛이 있으면
  발동을 보류(`reactionRequest`)하고 수비 측이 `react` 액션으로 `block` 또는 통과를 선택한다.
- **성검** 키워드: 아군 용사에 부여. 자신을 대상으로 하는 지혜 조건 주문을 무효화하고 지략 +5.

---

## 지략 (cunning)

- **수비 자동 반응**: 상대가 지혜 조건 카드를 낼 때, 내 진영에 조건량 이상의 지략을 가진 유닛이 있으면 봉쇄 선택권이 생긴다.
- 봉쇄하면 해당 유닛의 지략 1 소진, 카드 이번 턴 잠금.
- 반응 대기 중에는 다른 액션이 불가하다.

---

## 강제 능력 (자동 발동 효과)

조건이 갖춰지면 **자동으로 발동**되는 효과. 각 행동 후 정산된다.

| 트리거 | 예시 카드 |
|--------|----------|
| 내 전장이 비었을 때 | 복수자: 패에서 자동 소환 |
| 아군 유닛이 사망할 때 | 킹슬라임: 죽은 슬라임 수만큼 강화 |
| 이 유닛이 사망할 때 (`최후`) | 해골병사→해골 진행, 마왕→즉시 패배 |
| 내 진영 힘+지혜 최고일 때 | 배신자: 무작위 아군 파괴 후 상대 진영 이동 (1회) |
| 의식이 5회 수행되었을 때 | 마왕: 강림 (1회) |
| 환경이 변화할 때 | 용사: 환경 변화마다 강화 |
| 내 턴 시작 시 | 미후왕: 매 턴 패악질 발동 |
| 용사 레벨업 시 | 전사/사제/마법사: 레벨업 보너스 |
| 자기 전장에 머리 없을 때 | 목없는기사: 스스로 파괴 |

`once` 트리거는 1회만 발동된다 (`firedForced` 기록).

---

## 영웅담 (용사 유닛)

`영웅담` 키워드를 가진 유닛은 **레벨업** 시스템을 갖는다.

- 적 유닛을 파괴할 때 해당 유닛의 **힘+지혜** 점수가 누적된다.
- 누적 점수가 피보나치 임계치를 돌파할 때마다 **레벨업** → +1/+1 강화 + `heroLevelUp` 이벤트.
- **결속** 키워드: 한 턴에 결속 카드를 한 장만 낼 수 있다 (`bondPlayedThisTurn`).
- **액티브 능력** (`meta.activeAbility`): 공격 대신 특수 능력 사용 (`ability` 액션). 행동권 소모.
  - 사제: 아군 유닛 힘 부여
  - 마법사: 적 유닛 힘 감소

---

## 묘지 (graveyard)

- 유닛이 파괴되면 스냅샷이 **묘지**(`state.graveyard`)에 보관된다.
- `dead` 배경 조건으로 묘지의 카드를 참조할 수 있다.
- **교회**: 묘지의 아군 용사를 부활. 강화된 스탯·레벨·expMax 유지 (exp는 0 리셋).
- **이탈**(`exitUnit`) 처리된 유닛은 묘지에 남지 않는다 (`최후` 효과도 없음).

## 효과 어휘

| 어휘 | 의미 |
|------|------|
| **전개** | 환경에 타입:값을 추가/갱신 |
| **진행** | 유닛을 진화형으로 변신 |
| **강림** | 소환 불가 유닛을 특별 소환 |
| **이탈** | 사망 처리 없이 게임에서 퇴장 (최후 효과 없음, 묘지 안 감) |
| **개입** | 낸 즉시 효과 처리 |
| **결속** | 턴당 1장 제한 키워드 |
| **부동** | 이번 턴 능동 플레이어가 아무 행동도 안 했을 때만 효과 발동 |

## 진행 예시

```
[오프닝]
A: 돌원숭이 배치
B: 퀘스트-슬라임토벌 배치  ← 전개 효과 보류
```
