# 게임 규칙

이 문서는 **게임 엔진의 공통 규칙**만 다룬다. 개별 카드의 정확한 스탯·조건·효과는
게임 내 카드 설명(호버/줌)이나 소스(`src/rules/cards/defs/*.ts`)가 항상 최신 기준이다
— 이 문서에는 특정 카드를 예시로 들되, 그 카드의 스탯/효과 수치를 단정하지 않는다.

## 기본 구조

- **덱**: 카드 15장. 게임 시작 시 **전부 패**에 들고 시작한다. 필드는 비어 있다.
- **비용 없음**: 카드를 내는 데 별도의 자원(마나 등)이 필요하지 않다.
- **패배 조건**: **턴 종료 시 필드가 비어 있으면** 진다. 양측이 동시에 비면, 그
  턴을 끝낸(패스를 선언한) 쪽이 진다.

---

## 진행 순서

### 오프닝 페이즈

- 양측이 번갈아 가며 **최대 3장**을 필드에 낼 수 있다. 각 카드는 배치할 **셀 번호**를 지정한다.
- 유닛은 즉시 필드에 배치되어 이후 카드의 **배경 조건**을 충족시킬 수 있다.
- 오프닝 중 발동되는 **전개·효과는 보류**된다 — 양측이 모두 오프닝을 마칠 때 A→B 순서로 일괄 처리된다.
- 3장을 모두 내거나 `finishOpening`을 선언하면 오프닝 종료.
- 양측 오프닝이 끝나면 보류 효과 처리 → 강제 능력 정산 → **A 선턴 메인 페이즈** 시작.

### 메인 페이즈

- 한 턴에 **원하는 만큼** 카드를 낼 수 있다 (제한 없음).
- 카드 효과(`onPlay`)는 **턴 종료(pass) 시** 낸 순서대로 일괄 처리된다.
  - **`개입` 키워드**를 가진 카드만 예외 — 낸 즉시 효과가 처리된다. 개입 키워드가
    없는 주문은 유닛의 소환 효과와 마찬가지로 **패스 시점까지 미뤄진다.**
  - 유닛 소환(필드 배치) 자체는 항상 즉시 이루어진다 — 지연되는 건 부수 효과뿐.
- **각 유닛은 공격 또는 이동 중 하나만** 가능하다 (`actedThisTurn`).
- **패스**는 "이번 턴 끝"을 선언하는 행동이다. 카드를 내지 않고 바로 패스해도 된다.

## 전장 그리드

플레이어당 **9칸**: **전열**(0–4, 5슬롯) + **후열**(5–8, 4슬롯). 셀당 유닛 1개 최대.

```
전열:  [0] [1] [2] [3] [4]
후열:  [5] [6] [7] [8]
```

**인접 관계** (이동·협공 판정):

```
0↔1,5 / 1↔0,2,5,6 / 2↔1,3,6,7 / 3↔2,4,7,8 / 4↔3,8
5↔0,1,6 / 6↔1,2,5,7 / 7↔2,3,6,8 / 8↔3,4,7
```

**공격 범위** (기본, 상대 전열 대상):

| 공격 셀 | 대상 셀 |
|---------|---------|
| 전열 0 | 0, 1 |
| 전열 1 | 0, 1, 2 |
| 전열 2 | 1, 2, 3 |
| 전열 3 | 2, 3, 4 |
| 전열 4 | 3, 4 |
| 후열 5 | 0, 1 |
| 후열 6 | 1, 2 |
| 후열 7 | 2, 3 |
| 후열 8 | 3, 4 |

**이동**: 행동권(`actedThisTurn`)이 남은 유닛이 **인접 빈 셀**로 이동. 이동도 행동권 소모.

## 카드의 종류

| 종류 | 설명 |
|------|------|
| **유닛** | 필드에 배치되어 전투에 참가한다. 힘·지혜·지략을 가진다. 소환은 항상 즉시. |
| **주문** | 필드에 남지 않는다. `개입` 키워드가 있으면 낸 즉시 효과 발동, 없으면 유닛의 부수 효과와 동일하게 **턴 종료(pass) 시** 발동. |

---

## 능력치 (스탯)

| 스탯 | 이름 | 설명 |
|------|------|------|
| **힘** | 전투력 | 전투 시 높은 쪽이 낮은 쪽을 파괴. 동점이면 양쪽 모두 파괴. |
| **지혜** | 임계치 조건 | 소모되지 않음. 진영의 지혜 합계(또는 특정 유닛의 지혜)가 조건을 충족해야 특정 카드를 낼 수 있음. |
| **지략** | 반응 임계치 | 상대가 지혜 조건 카드를 낼 때, 조건량 이상의 지략을 가진 내 유닛으로 자동이 아닌 **선택적** 봉쇄 가능. |

스탯은 효과에 의해 **변동 가능**하다 (강화·교환·일시 버프 등).

---

## 배경 (배경 조건)

카드를 낼 때 **필요한 조건**. 내는 순간에만 확인하며, 이후 조건을 잃어도 이미
낸 카드는 무효화되지 않는다. 조건은 기본적으로 **자기 진영** 기준이지만, 카드에
따라 상대 진영을 기준으로 할 수도 있다.

| 조건 종류 | 예시 |
|-----------|------|
| 특정 유닛 존재 | 필드(양측 어디든)에 특정 이름의 유닛이 있어야 한다 |
| 키워드 보유 유닛 | 특정 키워드를 가진 유닛이 존재해야 한다 |
| 환경 항목 존재 | 환경에 특정 type:value 항목이 있어야 한다 |
| 진영 지혜 임계치 | 해당 진영의 지혜 합계 ≥ N |
| 단일 유닛 지혜 임계치 | 해당 진영의 유닛 중 하나의 지혜 ≥ N |
| 진영 힘 임계치 존재 | 해당 진영에 힘 N 이상인 유닛이 있어야 한다 |
| 강한 유닛 없음 | 해당 진영에 힘 N 이상인 유닛이 없어야 한다 |
| 묘지 조건 | 묘지에 특정 키워드를 가진 카드가 있어야 한다 |

---

## 환경 (환경 전개)

- 카드가 **전개** 효과로 환경을 바꿀 수 있다 (예: 지형:산, 지역:왕성 등).
- **같은 타입의 환경은 중복되지 않는다** — 새 값이 덮어쓴다.
- 서로 다른 타입은 동시에 공존한다. 타입은 고정된 목록이 아니라 임의 문자열이다.
- 환경은 **전역(글로벌)**으로 공유된다 — 양측이 같은 환경을 읽는다.

## 전투

### 1:1 전투 (기본)

1. 공격 유닛과 수비 유닛의 **힘을 비교**한다.
2. **힘이 낮은 유닛이 파괴**된다.
3. **힘이 같으면 양쪽 모두 파괴**된다.

### 협공 (다수 수비)

수비 측은 공격 선언에 대해 **여러 유닛으로 함께 방어**할 수 있다.

1. 공격자가 **공격 대상 유닛**을 지정한다.
2. 협공 가능한 인접 아군이 있으면 즉시 해결되지 않고, 수비 측이 **주 수비 유닛의
   인접 셀** 유닛을 추가 협공자로 지정할지 선택한다.
3. 협공 참여 유닛 전체의 **힘을 합산**해 공격자의 힘과 비교한다.
4. **수비 합산 ≥ 공격자** → 공격자·수비 유닛 **전원 생존** (공격 막힘, 공격자는
   협공으로는 파괴되지 않는다 — 의도된 설계)
5. **수비 합산 < 공격자** → 협공에 참여한 수비 유닛 **전원 파괴**, 공격자 생존

> 협공에 참여한 유닛은 **그 턴 동안 다른 공격에 협공 참여 불가** (`blockedThisTurn`,
> `이중방어` 키워드가 있으면 2회까지). 단, 자신이 공격 대상이 되는 것은 별개로 가능하다.

> `공격 불가` 키워드를 가진 유닛은 공격에 나설 수 없다.

> `협력불가` 키워드를 가진 유닛은 협공 블로커가 될 수도, 협공 방어를 받을 수도 없다.

> `전투면역` 키워드를 가진 유닛은 전투로 파괴되지 않는다 (효과에 의한 파괴는 별개로 적용됨).

### 타겟팅 리액션

공격·주문이 대상을 결정할 때 다음 파이프라인을 거친다:

- **호위** 키워드: 수비 측에 해당 키워드를 가진 유닛이 있으면, 원래 지정된 대상 대신
  다른 무작위 아군이 대상이 된다.
- **지략 opt-in**: 지혜 조건이 있는 카드가 플레이될 때 봉쇄 가능한 지략 유닛이 있으면
  발동을 보류하고 수비 측의 선택(봉쇄/통과)을 기다린다. 아래 "지략" 절 참고.
- **성검** 키워드: 아군 유닛에 부여되면, 자신을 대상으로 하는 지혜 조건 주문의
  지략 판정을 무효화한다.

---

## 지략 (cunning)

- 상대가 지혜 조건이 있는 카드를 낼 때, 내 진영에 조건량 이상의 지략을 가진
  유닛이 있으면 **봉쇄할지 선택**할 수 있다 (자동이 아니다 — 항상 선택지로만 주어짐).
- **봉쇄를 선택하면**: 그 카드는 이번 턴 동안 다시 낼 수 없게 잠기고(패에는 남는다),
  사용한 유닛은 이번 턴 다시 봉쇄에 쓸 수 없게 된다. **지략 수치 자체는 줄지 않으며**,
  다음 턴이 되면 다시 봉쇄에 쓸 수 있다.
- **통과를 선택하면**: 보류됐던 카드 효과가 정상적으로 발동한다.
- 반응 대기 중에는 다른 액션이 불가하다.

---

## 강제 능력 (자동 발동 효과)

조건이 갖춰지면 카드 효과와 무관하게 **자동으로 발동**되는 효과. 매 행동 뒤 정산된다.
크게 두 가지 방식이 있다:

- **이벤트 트리거**: 특정 사건(유닛 사망, 턴 시작, 환경 변화, 레벨업 등)이 발생한
  순간 한 번 발동한다.
- **정적 조건**: 매 정산마다 조건을 다시 검사해, 조건이 참인 동안(또는 참이 되는
  순간) 발동한다. 조건이 사라지면 더 이상 발동하지 않는다.

`once`로 표시된 강제 능력은 한 게임에서 **단 한 번만** 발동한다.

---

## 영웅담 (레벨업 유닛)

`영웅담`류 유닛은 레벨업 시스템을 갖는다.

- 적 유닛을 파괴할 때 해당 유닛의 **힘+지혜** 점수가 누적된다.
- 누적 점수가 정해진 임계치를 돌파할 때마다 **레벨업**하며, 레벨업 이벤트를 구독한
  다른 카드(동료 유닛 등)를 함께 강화시킬 수 있다.
- **결속** 키워드: 한 턴에 결속 카드를 한 장만 낼 수 있다.
- **액티브 능력**: 일부 유닛은 공격/이동 대신 특수 능력을 쓸 수 있다 — 행동권을
  대신 소모하며, 효과는 카드마다 다르다(아군 강화, 적 약화 등).

---

## 묘지 (graveyard)

- 유닛이 파괴되면 스냅샷이 **묘지**에 보관된다.
- 배경 조건으로 묘지의 카드를 참조할 수 있다("묘지에 특정 키워드 카드가 있어야 함").
- 부활 계열 카드는 묘지의 유닛을 되살릴 수 있다(강화된 스탯은 유지되고, 경험치 등
  진행도만 초기화되는 것이 일반적 패턴 — 정확한 규칙은 카드마다 다름).
- **이탈** 처리된 유닛은 묘지에 남지 않는다(사망 효과도 발동하지 않는다).

---

## 효과 어휘

| 어휘 | 의미 |
|------|------|
| **전개** | 환경에 타입:값을 추가/갱신 |
| **진행** | 유닛을 다른 카드(대개 상위 진화형)로 교체하되 능력치는 유지 |
| **강림** | 평소 손패에서 낼 수 없는(`소환 불가`) 유닛을 특별한 조건으로 소환 |
| **이탈** | 사망 처리 없이 게임에서 퇴장 (최후 효과 없음, 묘지에도 안 감) |
| **개입** | 낸 즉시 효과 처리 (없으면 턴 종료 시 처리) |
| **결속** | 턴당 1장 제한 키워드 |
| **부동** | 이번 턴 아무도 행동(공격/이동)하지 않았을 때만 효과 발동 |

## 진행 예시 (개념 흐름 — 실제 카드명은 예시일 뿐)

```
[오프닝]
A: 유닛 카드 배치
B: 배경 조건을 발생시키는 주문 카드 배치  ← 전개 효과는 보류됨
A: 다른 유닛 카드 배치
B: finishOpening 선언
→ 양측 오프닝 완료 → 보류 효과 일괄 처리(A→B) → 강제 능력 정산 → 메인 페이즈 시작 (A 선턴)

[메인 페이즈, A 턴]
A: 환경을 바꾸는 주문 카드 플레이 (개입 키워드 없음 → 효과는 지금 발동하지 않음)
A: 그 환경을 배경 조건으로 요구하는 유닛 카드 플레이
   → 아직 환경이 바뀌지 않았으므로, 조건을 만족 못 하면 낼 수 없다
     (개입 카드가 아닌 한, 같은 턴 안에서 "먼저 낸 카드의 효과"를 다음 카드의
     배경 조건으로 즉시 쓸 수는 없다 — 정산은 패스 시점)
A: pass
   → 큐에 쌓인 onPlay 효과들이 낸 순서대로 일괄 처리(환경 변경 등) → B 턴 시작

[메인 페이즈, B 턴]
B: 유닛으로 공격 선언 → (협공 가능한 수비가 있으면 A의 반응을 기다림) → 전투 해결
B: pass → A 턴
```

이 예시는 **개입 vs 지연 처리**, **배경 조건은 낼 때만 확인** 같은 규칙의 상호작용을
보여주기 위한 것이며, 특정 카드의 실제 효과를 나타내지 않는다.
